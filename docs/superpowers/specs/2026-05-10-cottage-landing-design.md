# Дизайн-спецификация: лендинг ВЕР СТРОЙ /cottage

**Дата:** 2026-05-10
**Автор:** brainstorming session
**Статус:** на ревью пользователем

## 1. Цель

Продающий лендинг под рекламный трафик (Яндекс.Директ + VK Ads) для привлечения частных заказчиков на строительство премиум-коттеджей от 200 м². Главная страница сайта остаётся как есть.

**Бизнес-цель:** заявка на расчёт сметы коттеджа от частного заказчика премиум-сегмента.

**Ключевая метрика:** конверсия посетитель → заявка с email/телефоном.

## 2. Аудитория и нарратив

**Аудитория:** частные заказчики, премиум-сегмент. Дома 200+ м², бюджет 15 млн+. Уральский регион (Пермь, Екатеринбург, Челябинск, Тюмень). Возраст 35–55, владельцы бизнеса или топ-менеджеры. Главные страхи: сорвут сроки, раздуют смету, бросят коробку, сделают «как все».

**Главный оффер:** *«Прочность торгового центра. В вашем доме.»*

**Нарративный приём:** ВЕР СТРОЙ строит ЖК, торговые центры, объекты ЛУКОЙЛ — теперь применяет ту же опалубку, бригады и инженерный контроль для частных коттеджей. Премиум-заказчик получает компанию, у которой опыт промышленных объектов, а не дачный шабашник.

**Доказательная база:** 15+ реализованных коммерческих объектов (ТЦ Вера, Елькина 12, ЛУКОЙЛ, Озон, ЖК Good Style, ФОК, онкологический центр). 0 брошенных строек. 6+ лет на рынке.

## 3. Архитектура

**Маршрут:** `/cottage` — новая страница в Next.js App Router.

**Файловое размещение:**
- `src/app/cottage/page.tsx` — серверный компонент страницы
- `src/app/cottage/layout.tsx` — отдельный layout с метаданными лендинга (опционально, если отличается от глобального)
- `src/app/cottage/components/` — секции, специфичные для лендинга (не переиспользуемые)
- `src/components/` — переиспользуемые элементы (кнопки, формы) добавляются сюда

**SEO/мета:**
- `<title>`: «Строительство коттеджей под ключ от 200 м² — ВЕР СТРОЙ, Пермь»
- meta description: акцент на опыт + сроки + фикс смета
- Open Graph: `og:image` — отдельный 1200×630 hero AI-фото
- JSON-LD: `Service` (тип услуги «Строительство коттеджей»), `LocalBusiness` (контакты ВЕР СТРОЙ)
- robots: index, follow
- Canonical: `https://верстрой.рф/cottage`

**Главная страница не модифицируется.** Лендинг живёт автономно.

**Backend:** переиспользуем существующий `POST /api/contact` (форма заявки уже подключена к Mail.ru SMTP + Telegram-боту). Калькулятор сметы — отдельный POST-эндпоинт `/api/cottage/quote` (создаём в этой задаче), который принимает параметры расчёта и отправляет email + Telegram. Капча — Yandex SmartCaptcha на форме заявки и калькуляторе.

**Зависимости:**
- Next.js + Tailwind (уже стоят)
- `framer-motion` (уже стоит, см. `MotionProvider.tsx`)
- Шрифты: Inter (есть), JetBrains Mono (добавить через `next/font/google`), display-шрифт PP Neue Montreal или Söhne — по бюджету: если PP Neue Montreal/Söhne купить нельзя, fallback на **Manrope** (бесплатный, geometric sans, близкий по характеру)
- Никаких новых runtime-библиотек

## 4. Дизайн-система

**Стиль:** D — инженерный blueprint. Светлый фон, чертёжная сетка, sans-serif с жёлтым акцентом, моноширинные подписи. Эстетика инженерного проекта, не «строительной компании».

### Цветовые токены

| Токен | Значение | Применение |
|---|---|---|
| `--bg` | `#fafaf7` | основной фон секций |
| `--ink` | `#0a0a0a` | основной текст |
| `--ink-soft` | rgba(10, 10, 10, 0.6) | вторичный текст |
| `--accent-yellow` | `#ffd400` | подсветка ключевых слов в заголовках, CTA hover |
| `--accent-orange` | `#ff5722` | секондари-CTA, индикаторы прогресса |
| `--grid` | rgba(10, 10, 10, 0.06) | blueprint-сетка |
| `--dark-bg` | `#0a0a0a` | финальный CTA, фото-overlay |
| `--dark-ink` | `#fafaf7` | текст на тёмном |

### Типографика

| Назначение | Шрифт | Размеры |
|---|---|---|
| Display (h1, h2) | Manrope 800 (или PP Neue Montreal если куплен) | h1: clamp(40px, 8vw, 96px), h2: clamp(32px, 5vw, 64px) |
| Body | Inter 400/500 | 16/18px |
| Mono / tag-метки | JetBrains Mono 400 | 11–12px, letter-spacing 0.2em, uppercase |

Все display-заголовки с letter-spacing -0.04em и line-height 0.92–0.95 (плотная типографика).

### Сетка и spacing

- 32px blueprint-сетка (`background-image` linear-gradient в фоне секций)
- Секции: padding-y `clamp(80px, 12vw, 160px)`, padding-x `clamp(20px, 5vw, 80px)`
- Контейнер: max-width 1440px

### Радиусы и линии

- Скругления: 0 или 4px max (брутализм)
- Бордеры: 1px solid rgba(0,0,0,0.08), штриховые `1px dashed` для секций-разделителей
- Кнопки: прямоугольные, без скруглений, чёрный фон / жёлтый hover

### Анимации

- `framer-motion` `whileInView` для reveal секций
- CountUp для метрик
- Scroll-driven для процесс-таймлайна (секция 06) — `useScroll` + `useTransform`
- Параллакс на hero-фото — лёгкий transform translateY
- Все анимации respect `prefers-reduced-motion`

## 5. Структура секций

11 секций. Все содержат tag-метку с номером (`01 / 11`, `02 / 11`...) в стиле архитектурного штампа.

### 01. Hero (Split-вариант H1)

**Layout:** двухколоночный grid 1.1fr / 1fr. На мобиле — фото сверху, текст снизу.

**Левая колонка** (на blueprint-сетке):
- Tag: `VER STROY · 01 / 11 · COTTAGE 200+ M²`
- H1: «Прочность торгового центра. В вашем доме.» (фразу «торгового центра» — выделяем жёлтой плашкой `--accent-yellow` 6px padding)
- Подзаголовок: «6 лет строим коммерческие объекты по Уралу. Та же опалубка, те же бригады, та же арматура — для вашего коттеджа.»
- CTA-кнопка: «Расчёт за 24 часа →» (тёмная, ведёт на форму в секции 11 со scroll-smooth)
- 3 метрики снизу: `15+ объектов`, `6 лет на рынке`, `0 брошено`

**Правая колонка:**
- AI-фото премиум-коттеджа в архитектурной фотографии (золотой час, монолит+дерево, тон бренда)
- Лёгкий параллакс при скролле
- Подпись в углу JetBrains Mono: координаты Перми / номер AI-фото

### 02. Сравнение «ТЦ ↔ дом»

Сплит-секция с двумя крупными изображениями.

**Левый кадр:** реальное фото колонн/монолитной коробки ТЦ Вера или Елькина 12 (берём из существующего портфолио).
**Правый кадр:** AI-фото монолитной коробки коттеджа в той же стадии.

Поверх каждого кадра — tag-метка с указанием объекта (`ТЦ ВЕРА · ВЕРЕЩАГИНО · 2024`, `КОТТЕДЖ 280 М² · ПЕРМЬ · 2025`).

Под кадрами одна крупная фраза: «Та же опалубка. Та же команда. Та же арматура.»

При наведении (или scroll-progress) — кадры приближаются друг к другу (motion).

### 03. Интерактивный blueprint дома

**Главный «вау»-элемент лендинга.**

SVG-чертёж коттеджа в плане + разрез, на blueprint-сетке. Чертёж кастомный (рисуем сами или адаптируем стоковый), не реальный архитектурный план — стилизованная схема.

При hover/tap по 5 узлам подсвечивается линия и выезжает мини-карточка с историей:
1. **Фундамент** — «Монолитная плита. Защитный слой по СП 63.13330.»
2. **Монолит** — «Несущие стены и перекрытия — собственная опалубка.»
3. **Кровля** — «Опыт ЛУКОЙЛ. Снеговая нагрузка по СП 20.»
4. **Фасад** — «Вентилируемые фасады, опыт 5+ объектов.»
5. **Инженерия** — «Своя бригада. Контроль 6 этапов.»

Каждая мини-карточка имеет ссылку «Смотреть пример →», которая ведёт в секцию портфолио на конкретный объект.

На мобиле — без hover, узлы раскрываются вертикальным аккордеоном.

### 04. Цифры-якоря

Полноширинная тёмная полоса (`--dark-bg`).

4 крупные метрики в ряд (на мобиле — 2×2):
- `15+` ОБЪЕКТОВ
- `8 200 м³` БЕТОНА УЛОЖЕНО
- `0` БРОШЕННЫХ СТРОЕК
- `6` ЛЕТ НА РЫНКЕ

Цифры в Manrope 800, подписи в JetBrains Mono. CountUp при появлении в viewport.

### 05. 5 принципов работы

Bento-grid 5 карточек (на десктопе — 3 в верхнем ряду + 2 в нижнем со смещением, на мобиле — стек).

Каждая карточка:
- Номер `№ 01`–`№ 05` в JetBrains Mono
- Заголовок-принцип (3–5 слов)
- Описание-эссе (60–80 слов)
- Микро-доказательство (число или факт)

**Принципы:**
1. **Опалубка как у промышленников** — собственная, не арендованная. Ровные стены, минимум доводки.
2. **Бригады, а не подрядчики** — постоянный штат, тот же мастер ведёт от нуля до ключей.
3. **Опыт коммерческих объектов** — те, кто заливал колонны ТЦ, заливают и фундамент вашего коттеджа.
4. **Фиксированная смета** — закрепляется в договоре. Доплаты только если меняете проект.
5. **Один прораб от нуля до ключей** — без перекидываний между сменами.

### 06. Процесс — горизонтальный таймлайн

7 этапов от проекта до ключей. Реализация: scroll-driven с `useScroll` + `useTransform`. По мере прокрутки секции прогресс-бар движется горизонтально, активный этап подсвечивается.

**Этапы:**
1. Проект и расчёт (1–2 нед)
2. Котлован и подготовка (2 нед)
3. Монолит фундамента (2–3 нед)
4. Коробка / стены (4–8 нед)
5. Кровля (2 нед)
6. Фасад и инженерия (4–6 нед)
7. Отделка и ключи (8–12 нед)

Сроки указываются для коттеджа 200–300 м² «под ключ», итоговый срок 6–9 месяцев в зависимости от сложности.

На мобиле — вертикальный таймлайн без scroll-trigger.

### 07. Портфолио — bento с фильтрами

Сетка-bento с реальными фото из `covers/` и `photos/`. Фильтры: «Все» / «Торговые центры» / «ЖК и многоквартирные» / «Промышленные» / «Жилые дома».

В первом запуске лендинга секции «Жилые дома» нет реальных объектов → секция временно скрыта или показывается с подписью «Первый коттедж — ваш». Решение: фильтр «Жилые дома» отключаем до появления первого реализованного коттеджа.

Карточка объекта:
- Фото
- Название (ТЦ Вера, ЖК Good Style и т.д.)
- Площадь / тип / год
- Tag-метка категории

Клик → лайтбокс с галереей фото объекта.

### 08. Калькулятор сметы

Интерактивный калькулятор с live-обновлением.

**Параметры ввода:**
- Площадь дома: слайдер 100–500 м² (шаг 10)
- Тип коробки: радио — Монолит / Газобетон / Кирпич
- Этажность: радио — 1 / 2 / 2+мансарда
- Отделка: радио — Под чистовую / Под ключ
- Инженерия: чек-боксы — Тёплый пол / Котельная / Канализация / Скважина

**Расчёт:** базовая стоимость м² по типу коробки × площадь × коэффициенты (этажность 1.0/1.15/1.25, отделка 0/+25%, инженерия суммой). Цены — задаются как константы в `cottage-pricing.ts`. Текущие ориентиры (актуализировать перед релизом по фактической смете прораба):
- Монолит: 60 000 ₽/м²
- Газобетон: 45 000 ₽/м²
- Кирпич: 55 000 ₽/м²

Live-выдача: «Ориентировочно: **9 600 000 ₽** · 12 месяцев · 280 м² монолит под ключ»
Дисклеймер: «Точная смета — после выезда инженера на участок».

CTA: «Получить детальную смету на email» — форма с email + телефон + подгрузкой расчёта в email.

Все цифры — ориентировочные. Точная смета считается прорабом.

### 09. Команда — лица прорабов

3 карточки ключевых прорабов (фото, имя, должность, опыт, какие объекты вёл).

Цель: «безликий бренд» → «вот человек, который будет вести стройку». Снимает страх перекидываний.

Реальные фото предоставляет ВЕР СТРОЙ. До получения фото — заглушки в стиле архитектурного штампа (силуэт в blueprint).

### 10. FAQ + договор

Аккордеон 6–7 вопросов:
1. Сколько стоит дом 200 м²?
2. Сколько идёт строительство?
3. Что входит в фикс-смету, а что нет?
4. Какие гарантии после сдачи?
5. Можно ли посмотреть текущие стройки?
6. Работаете в моём городе? (Пермь, Екатеринбург, Челябинск, Тюмень)
7. Кто отвечает за качество, если уволится прораб?

Кнопка «Скачать образец договора» → PDF с маскированными реквизитами (готовится отдельно через `make-contract`).

### 11. Финальный CTA

Тёмный полноширинный блок (`--dark-bg`) с blueprint-сеткой поверх в `--accent-yellow` низкой opacity.

Большой заголовок: «Расчёт сметы за 24 часа.»
Подзаголовок: «Бесплатно. Без обязательств. Подпишем NDA по запросу.»

Форма:
- Имя
- Телефон (обязательное)
- Email
- Площадь дома (опционально)
- Сообщение (опционально)
- Yandex SmartCaptcha
- Кнопка «Отправить заявку»

Рядом — крупные кнопки телефон / Telegram / WhatsApp.

После отправки → success-state с обещанием «Мы свяжемся в течение 24 часов».

## 7. Адаптивность

- Mobile (≤640px): один столбец, hero-фото сверху, scroll-таймлайн → вертикальный, blueprint-чертёж → аккордеон узлов
- Tablet (641–1024px): большинство секций сохраняют десктопный layout
- Desktop (1025+): полная композиция

Минимальная ширина — 360px.

## 8. Доступность

- WCAG 2.1 AA — контраст текст/фон ≥ 4.5:1 для body, ≥ 3:1 для заголовков
- Все интерактивные узлы blueprint доступны клавиатурой (`tabindex=0`, Enter/Space)
- `prefers-reduced-motion` отключает scroll-анимации
- alt-тексты на все AI-фото с описанием
- aria-label на CTA, формы, аккордеоны
- Лайтбокс портфолио — focus trap

## 9. Производительность

- Все AI-фото в `next/image` с placeholder=blur
- AVIF/WebP fallback
- Lazy-loading секций ниже первого экрана
- LCP target: ≤2.5s, CLS ≤0.1
- Размер бандла лендинга: ≤200 KB JS gzipped (без учёта next.js core)

## 10. Аналитика и трекинг

- Яндекс.Метрика (когда подключится — см. дашборд) — события: `cottage_form_submit`, `cottage_calc_used`, `cottage_phone_click`, `cottage_telegram_click`
- UTM-метки сохраняются в скрытые поля формы → передаются в email/Telegram

## 11. Что НЕ делаем (YAGNI)

- ❌ Видео-фон (тяжело, на мобиле провал)
- ❌ 3D-модель дома (overkill, blueprint SVG достаточен)
- ❌ Онлайн-чат (есть телефон + Telegram + форма)
- ❌ Блог/статьи на лендинге
- ❌ Личный кабинет / авторизация
- ❌ Калькулятор не делаем «точным до рубля» — только ориентировочные цифры
- ❌ Не интегрируем CRM на старте (заявки идут в email + Telegram, как уже работает)

## 12. Приёмка (acceptance criteria)

Лендинг считается готовым, когда:

1. Открывается по URL `/cottage`
2. Все 11 секций отрисовываются на десктопе и мобиле
3. Hero визуально соответствует утверждённому мокапу H1
4. Интерактивный blueprint (секция 03) работает: hover/tap раскрывает мини-карточки
5. Scroll-таймлайн (секция 06) реагирует на прокрутку
6. Калькулятор сметы (секция 08) выдаёт live-расчёт
7. Форма заявки (секция 11) отправляется в email + Telegram, проходит SmartCaptcha
8. Все AI-фото загружены и оптимизированы
9. SEO-мета и JSON-LD проставлены
10. Lighthouse Performance ≥ 90, Accessibility ≥ 95
11. `npm run build` проходит без ошибок
12. Прошло проверку `/design-check` (4 агента)

## 13. Риски

| Риск | Митигация |
|---|---|
| AI-фото получаются неубедительными | Итерируем промпты, при провале — фоллбек на премиум-сток |
| Нет реальных фото коттеджей | Честный нарратив «опыт ТЦ → теперь дома», секция «Жилые дома» в портфолио скрыта до первого объекта |
| Калькулятор даёт нереалистичные цифры | Калибруем по фактической смете прораба перед релизом |
| Цены за м² меняются | Хранятся константами в одном файле, легко обновить |
| Шрифт PP Neue Montreal/Söhne не куплен | Fallback на Manrope (бесплатный, geometric sans, близкий) |
| Mail.ru SMTP не работает (известная проблема из дашборда) | Лендинг готовим параллельно с настройкой SMTP_PASS, релиз после фикса |
