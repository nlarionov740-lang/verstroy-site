# 3D Terrain Animation — Design Spec

**Дата:** 2026-04-11
**Статус:** Утверждён

## Цель

Добавить на главную страницу сайта ВЕР СТРОЙ два 3D-эффекта:
1. **TerrainBackground** — полноэкранный анимированный 3D-рельеф за контентом (fixed фон)
2. **TerrainDivider** — анимированные горные разделители между секциями

Стиль: 3D-ландшафт из wireframe-линий, ассоциация со стройплощадкой и землёй.
Интенсивность: средняя-яркая — заметно, но контент читается.

## Технический стек

- **Three.js** через `@react-three/fiber` (R3F) + `@react-three/drei`
- **simplex-noise** для генерации рельефа
- **Framer Motion** — уже установлен, используется для scroll-значений
- **Canvas 2D** — для разделителей (легче чем отдельные Three.js сцены)

## Компонент 1: TerrainBackground

### Описание
Полноэкранный Canvas с Three.js сценой. Позиция `fixed`, покрывает весь viewport, находится за контентом (z-index: 0).

### Геометрия
- `PlaneGeometry` с сегментами 80x50 (desktop) / 30x20 (mobile)
- Wireframe-материал, без заливки
- Вершины смещаются по оси Z через simplex noise

### Анимация
- Непрерывная волна: вершины = `simplex3(x * freq, y * freq, time * speed) * amplitude`
- Амплитуда: 25-35 единиц
- Скорость: медленная, ~0.3 единицы/сек

### Интерактивность
- **Курсор:** Raycaster определяет позицию мыши на плоскости. Вершины в радиусе ~5 единиц от курсора получают дополнительное смещение вверх (15-20 единиц). Плавное затухание.
- **Скролл:** `window.scrollY` смещает фазу шума (scrollY * 0.001), создавая эффект движения рельефа при прокрутке.
- **Мобильные:** Без реакции на курсор (нет hover), только скролл-фаза.

### Визуал
- **Основные линии (каждая 3-я):** `#E4BE6A` (яркий золотой), opacity 0.5-0.7
- **Вторичные линии:** `#243558` (синий), opacity 0.2-0.3
- **Glow на вершинах:** Points с billboard-спрайтами, размер пропорционален высоте, цвет `#E4BE6A`, alpha зависит от высоты
- **Вертикальные линии-связки:** Тонкие (#D4A843, opacity 0.1), соединяют каждую 5-ю вершину с соседним рядом

### Камера
- `OrthographicCamera` или `PerspectiveCamera` с малым FOV (~35°), смотрит на плоскость под углом ~60° сверху
- Позиция камеры статична, двигается только геометрия

### Перфоманс
- Анимация через `useFrame` (R3F встроенный RAF)
- Vertex displacement в JS (не шейдер) для простоты
- FPS-монитор: если fps < 30 на первых 60 кадрах — переключить на статичный SVG-фон
- `devicePixelRatio` ограничен до 2

## Компонент 2: TerrainDivider

### Описание
Размещается между секциями (Hero→About, About→Services и т.д.). Высота 150-200px. Отдельный Canvas 2D для каждого разделителя.

### Визуал
- 6-8 горизонтальных волнистых линий, имитирующих горный хребет
- Первая (верхняя) линия — яркая (#E4BE6A, opacity 0.6, lineWidth 2), с заливкой-градиентом вниз
- Последующие — затухающие по яркости и толщине
- Glow-точки на пиках первой линии (radialGradient)

### Анимация
- Непрерывная волна через `sin/cos` комбинации
- Скролл смещает фазу (parallax-эффект)
- Скорость: 0.3-0.4

### Размещение
- Между секциями в `page.tsx`: Hero, About, Services, Portfolio, Process, Stats, Contacts
- Всего ~6 разделителей

## Интеграция в существующий сайт

### Новые файлы
- `src/components/TerrainBackground.tsx` — Three.js фоновая сцена
- `src/components/TerrainDivider.tsx` — Canvas 2D разделитель

### Изменения в существующих файлах
- `src/app/page.tsx` — добавить `<TerrainBackground />` и `<TerrainDivider />` между секциями
- `package.json` — добавить зависимости: `three`, `@react-three/fiber`, `@react-three/drei`, `simplex-noise`

### Что НЕ трогаем
- Hero-блок (видео-фон остаётся)
- Контент секций
- Навигацию
- Существующие анимации Framer Motion

## Мобильная адаптация

- Сетка: 30x20 сегментов (вместо 80x50)
- Без реакции на курсор
- Пониженная частота обновления (throttle до 30fps)
- Разделители: 4 линии вместо 8
- Breakpoint: `md` (768px) — порог для упрощения

## Fallback

Если WebGL недоступен или fps < 30:
- TerrainBackground → скрыть Canvas, оставить текущие CSS-градиенты
- TerrainDivider → статичный SVG с волнистыми линиями (без анимации)
